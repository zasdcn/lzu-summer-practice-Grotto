# 网站部署指南

本指南提供多种方式将您的兰州大学暑期实践团网站部署到可以让其他人访问的环境中。

## 方案一：本地网络部署（局域网访问）

### 适用场景
- 同一WiFi网络下的用户访问
- 办公室或学校内部网络访问
- 临时演示或测试

### 部署步骤

1. **启动应用**
   ```bash
   python app.py
   ```

2. **获取本机IP地址**
   - Windows: 打开命令提示符，输入 `ipconfig`
   - 查找 "IPv4 地址" 或 "IP Address"，例如：192.168.1.100

3. **访问网站**
   - 本机访问：http://localhost:5000
   - 其他设备访问：http://你的IP地址:5000
   - 例如：http://192.168.1.100:5000

### 注意事项
- 确保防火墙允许5000端口访问
- 所有访问者必须在同一网络内

## 方案二：云平台部署（推荐）

### 2.1 Vercel部署（免费，推荐）

**优点：** 免费、快速、支持自动部署

**步骤：**

1. **准备代码**
   - 在项目根目录创建 `vercel.json` 配置文件
   - 创建 `requirements.txt` 文件

2. **注册Vercel账号**
   - 访问 https://vercel.com
   - 使用GitHub账号注册

3. **上传到GitHub**
   - 创建GitHub仓库
   - 上传项目代码

4. **连接Vercel**
   - 在Vercel中导入GitHub项目
   - 自动部署完成

### 2.2 Heroku部署

**优点：** 稳定、支持数据库

**步骤：**

1. **安装Heroku CLI**
   - 下载：https://devcenter.heroku.com/articles/heroku-cli

2. **创建应用**
   ```bash
   heroku create 你的应用名称
   ```

3. **部署**
   ```bash
   git push heroku main
   ```

### 2.3 PythonAnywhere（免费额度）

**优点：** 专为Python设计，简单易用

**步骤：**
1. 注册 https://www.pythonanywhere.com
2. 上传代码文件
3. 配置Web应用
4. 启动服务

## 方案三：内网穿透（临时方案）

### 3.1 使用ngrok

**优点：** 快速、无需配置服务器

**步骤：**

1. **下载ngrok**
   - 访问 https://ngrok.com
   - 下载并注册账号

2. **启动应用**
   ```bash
   python app.py
   ```

3. **创建隧道**
   ```bash
   ngrok http 5000
   ```

4. **获取公网地址**
   - ngrok会提供一个临时的公网地址
   - 例如：https://abc123.ngrok.io

### 3.2 使用花生壳（国内）

**步骤：**
1. 下载花生壳客户端
2. 注册账号并实名认证
3. 配置内网穿透
4. 获取公网域名

## 方案四：自己的服务器部署

### 4.1 云服务器（阿里云、腾讯云等）

**步骤：**

1. **购买云服务器**
   - 选择Linux系统（推荐Ubuntu）
   - 配置安全组，开放5000端口

2. **安装环境**
   ```bash
   sudo apt update
   sudo apt install python3 python3-pip
   pip3 install flask
   ```

3. **上传代码**
   ```bash
   scp -r /path/to/your/project user@server_ip:/home/user/
   ```

4. **运行应用**
   ```bash
   cd /home/user/your-project
   python3 app.py
   ```

5. **配置域名**（可选）
   - 购买域名
   - 配置DNS解析到服务器IP

### 4.2 使用Nginx反向代理（生产环境推荐）

**配置示例：**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 推荐方案总结

| 方案 | 适用场景 | 成本 | 难度 | 稳定性 |
|------|----------|------|------|--------|
| 本地网络 | 临时演示 | 免费 | 简单 | 一般 |
| Vercel | 静态展示 | 免费 | 简单 | 高 |
| Heroku | 完整功能 | 免费额度 | 中等 | 高 |
| ngrok | 临时分享 | 免费额度 | 简单 | 一般 |
| 云服务器 | 生产环境 | 付费 | 复杂 | 很高 |

## 快速开始建议

1. **立即可用：** 使用方案一（本地网络）
2. **长期使用：** 推荐Vercel或Heroku
3. **临时分享：** 使用ngrok
4. **专业部署：** 云服务器 + Nginx

---

## 需要帮助？

如果在部署过程中遇到问题，可以：
1. 检查防火墙设置
2. 确认端口是否被占用
3. 查看应用日志
4. 联系技术支持

**注意：** 部署到公网时，请确保数据安全，建议添加用户认证和HTTPS支持。
